## Анализ промтов (эффективность)

### Рабочие паттерны
- Промт с явной целью + порядком шагов + ожидаемыми артефактами давал стабильный результат.
- Промт с жёстким порядком merge-пакетов (`docs -> tooling -> core`) снижал регрессии.
- Промт с требованием precheck перед каждым git-шагом предотвращал ложные действия вне git-root.

### Слабые паттерны
- Общий промт «продолжай» без фиксированных критериев вызывал разрастание служебных артефактов.
- Промт без требования non-interactive режима неявно допускал terminal retry loops.

### Улучшенный шаблон промта для операций
1. Цель и критерий готово (измеримый).
2. Порядок шагов с precheck перед mutation.
3. Явный список обязательных quality/governance проверок.
4. Список обязательных артефактов (пути и формат).
5. Правило fallback: при блокере — fail-fast диагностировать и продолжать без интерактива.
