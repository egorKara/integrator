# Routing Policy для 24/7 AI-комбайна (2026-03-05)

## Цель
- Максимизировать uptime и полезность ответа при минимальном budget burn.
- Сохранить качество за счёт selective escalation вместо “всегда дорогая модель”.

## Классы задач
- **T0 Trivial**: короткие справки, статус, шаблонные ответы.
- **T1 Standard**: обычные продуктовые/операционные вопросы, легкий анализ.
- **T2 Complex**: архитектура, нестандартный дебаг, многокритериальные решения.
- **T3 Critical**: блокеры продакшена, риск потери данных/доступа.

## Роутинг по умолчанию
1. **Primary**: LM Studio (локальная маломодель) для T0/T1.
2. **Fallback A**: OpenRouter free/low-cost модель для T1/T2, если local degraded.
3. **Fallback B**: Groq/HF route для burst-ответа (в рамках лимитов).
4. **Premium Escalation**: Codex только для T2/T3 и только по trigger-правилам.

## Trigger-правила эскалации в Codex
- Два последовательных неуспешных ответа на ту же задачу.
- Выявлен высокий риск регрессии/поломки критичного потока.
- Запрос явно tagged как `critical` или `high-complexity`.
- Доступен суточный budget Codex выше порога.

## Anti-burn guardrails
- Дневной бюджет Codex: фиксированный ceiling + hard stop.
- Почасовой лимит эскалаций: `N` задач/час.
- При достижении лимита: автоматический downgrade на fallback с явным уведомлением.
- Для free providers: circuit-breaker после 3 подряд 429/5xx.

## Деградационные режимы
- **Degraded-1**: local доступен, внешние недоступны -> отвечаем локально + ограниченный режим.
- **Degraded-2**: local недоступен, внешние лимитированы -> короткий safe-response + issue-лог + retry.
- **Degraded-3**: все контуры недоступны -> аварийное уведомление в Telegram + GitHub issue.

## Политика данных
- По умолчанию во внешние API отправляется только минимально необходимый контекст.
- Секреты, токены, приватные артефакты и raw dumps запрещены к отправке.
- Для sensitive-кейсов внешние провайдеры отключаются принудительно.

## Метрики успеха policy
- p95 first-response latency.
- Доля успешных ответов без эскалации.
- Codex burn/day.
- Доля деградационных ответов.
- Mean-time-to-recover при provider outage.
