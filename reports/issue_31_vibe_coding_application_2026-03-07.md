# Issue #31 — Vibe-coding: выжимка и применение

## Выжимка по материалу
- Vibe-coding фокусируется на качестве взаимодействия человек↔AI, а не на объёме ручного кода.
- Практический паттерн: быстрое свободное описание идеи -> структурирование в проверяемые шаги -> короткие циклы валидации.
- “Skill как именованный контекст” повышает повторяемость качества и снижает стоимость переключения контекста.

## Что применено в проекте
- Добавлены специализированные skills под GitHub review:
  - `.trae/skills/github-pr-reviewer/SKILL.md`
  - `.trae/skills/github-security-reviewer/SKILL.md`
- Добавлена явная привязка этих review skills в `docs/CODE_REVIEW.md`.
- Индекс документации обновлён ссылками на новые skills.

## Почему это повышает продуктивность
- Снижается время на “объяснение с нуля” ревью-подхода.
- Повышается консистентность PR-review между разными сессиями.
- Уменьшается риск пропуска критичных security и contract check пунктов.

## Рекомендованный операционный цикл
1) Intake задачи в 3-5 буллетов (идея/ограничения/критерий done).
2) Быстрый выбор нужного skill-контекста.
3) Реализация крупным блоком.
4) Валидация через preflight + quality gates.
5) Фиксация артефактов и выводов в issue/report.
