# Network Adapter RCA — 2026-03-08

## Подтверждённые факты
- В `C:\Windows\INF\setupapi.dev.log` есть повторяющиеся ручные удаления Realtek NIC через Device Manager:
  - `2026/02/20 10:43:22`
  - `2026/02/24 14:14:11`
  - `2026/03/08 17:22:55`
- После удаления/переобнаружения адаптер возвращается и связь восстанавливается.
- В интервале инцидента присутствуют множественные DNS timeout события (`Microsoft-Windows-DNS-Client`, Event ID 1014).
- В тот же период зафиксированы операции удаления Wintun-устройств от `HitVPN.exe` в setupapi (`Delete Device - SWD\WINTUN\...`).
- На Realtek адаптере включены энергосберегающие параметры (`Energy Efficient Ethernet`, `Green Ethernet`, `Gigabit Lite`, `Power Saving Mode`).

## Гипотеза root-cause
- Наиболее вероятен комбинированный эффект:
  1) нестабильный профиль энергосбережения Realtek;
  2) агрессивная ротация виртуальных Wintun-адаптеров VPN-клиентом;
  3) каскадные DNS timeouts при флапах.

## Практическая стратегия
- Закрепить стабильный профиль Realtek (отключить энергосбережение NIC).
- Исключить churn Wintun (ограничить параллельные VPN-сессии, обновить/переставить VPN-драйвер).
- Фиксировать каждый инцидент артефактами до/после (driver state + DNS + SSH reachability).

## Готовый инструмент
- `C:\integrator\network_stability_hardening.ps1`:
  - применяет стабильные параметры NIC;
  - перезапускает адаптер;
  - делает контрольную проверку `192.168.31.124:22`;
  - сохраняет лог в `C:\integrator\reports\nic_stability_hardening_*.log`.
