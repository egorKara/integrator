# Приоритеты задач проекта (2026-02-20)

## P0 (критично: целостность данных/памяти)

- Надёжная запись в память агента из файлов (без усечений/ошибок JSON)
  - Зависимости: RAG server доступен; при включённом RAG_AUTH нужен токен; при AGENT_WRITE_ENABLED=0 запись запрещена.
  - Артефакт: `python -m integrator localai assistant memory-write --content-file ...` (табличный/JSON вывод статусов HTTP и record.id).

- Атомарная запись отчётов/артефактов, чтобы исключить частичную перезапись
  - Зависимости: нет.
  - Артефакт: `_write_text_atomic()` используется в генераторах отчётов.

## P1 (важно: качество и сопровождение)

- Поднять покрытие `utils.py` целевыми unit-тестами
  - Зависимости: не блокирует P0; снижает риск регрессий при изменениях CLI и утилит.
  - Цель: покрыть атомарную запись и обработку ошибок путей.

- Стандартизировать операционный поток “проверка -> запись памяти -> отчёт”
  - Зависимости: P0 выполнен.
  - Артефакты: `OPERATIONS_QUICKSTART.md`, `reports/quality_report_2026-02-20.md`, `reports/audit_conclusion_2026-02-20.md`.

## P2 (планово: архитектура и удобство)

- Дальнейшая декомпозиция `cli.py` по подсистемам (без изменения поведения)
  - Зависимости: P0/P1 не блокируются, но улучшает скорость изменений.

- Автоматизация preflight-проверок (doctor/diagnostics/agents status) в одном сценарии
  - Зависимости: стабильные quality gates.
