# Пояснительная записка по спорным решениям (2026-02-20)

## T/A/C-001: `.tmp/testtmp/*` (ACL-аномалии, Permission denied)

1) Тезис (T): удалять/чинить ACL у `.tmp/testtmp/*`, чтобы Git и статические анализаторы могли сканировать дерево без предупреждений.
2) Антитезис (A): не трогать ACL (требует админ-прав/владения, риск непреднамеренного повреждения локальных артефактов), а сделать исключения в инструментах и Git.
3) Синтез (C): исключить `.tmp/` из Git и из ruff/mypy, чтобы устранить недетерминированность проверок без изменений ACL.
4) Документация рассуждений:
   - За (T): “чистая” проверка дерева, меньше скрытых проблем.
   - Против (T): на Windows исправление ACL может быть разрушительным и потребовать прав администратора.
   - За (A): безопасно и переносимо, устраняет шум и делает проверки воспроизводимыми.
   - Против (A): ACL-проблема физически остаётся на диске.
   - Выбор (C) соответствует требованию portable/без админ-прав и даёт детерминированный quality gate.
