# Memory Task (T+A=S) — безопасный режим для `obsidian eval`

Связанные материалы:
- Obsidian CLI help (`eval`): https://help.obsidian.md/cli

## Тезис (ситуация)
- `obsidian eval` позволяет выполнить JavaScript внутри приложения Obsidian.
- Это полезно для health-check и диагностики (например, количество файлов в vault).

## Антитезис (проблема/риски/неизвестное)
- `eval` открывает риск выполнения произвольного кода и не должен быть доступен без явного включения.
- В интеграциях LocalAI недопустим путь “передать произвольный JS из запроса пользователя”.

## Синтез (решение/подход)
- Добавить `integrator obsidian eval` как отключенную по умолчанию команду:
  - включение через явный флаг `--enable-eval`
  - выполнение только заранее заданных “профилей” (`--profile files_count`, `--profile active_file_path`, …)
  - запрет параметра `--code` и любых свободных строк
- Логирование:
  - не печатать код/контент
  - сохранять только факт выполнения профиля и длину результата

## Верификация (Planned)
- Unit-тесты: при отсутствии `--enable-eval` команда возвращает `status=disabled`.
- Smoke: `python -m integrator obsidian eval --enable-eval --profile files_count --json`
