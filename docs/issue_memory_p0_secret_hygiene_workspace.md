# Memory Task (T+A=S)

## Тезис (ситуация)
- В `C:\integrator` рядом с кодом живут чувствительные файлы `.env` и большие артефактные деревья, которые игнорируются Git, но остаются в файловой системе.

## Антитезис (проблема/риски/неизвестное)
- Это повышает риск случайных утечек через бэкапы/шары/скриншоты/ручные архивы и усложняет “чистый” анализ агента.
- Нельзя ломать текущий операторский workflow (нужны удобные секреты и артефакты).

## Синтез (решение/подход)
- Зафиксировать минимальный безопасный протокол:
  - секреты остаются локально (в `.env`/token-file), не выводятся в отчётах;
  - добавить “санитарный чек” на наличие секрет-паттернов в `reports/` и в отслеживаемых файлах перед публикацией артефактов;
  - документировать рекомендованное хранение артефактов (вне репо либо через отдельные корни).

## Контекст
- Репо/модуль: `integrator` (security hygiene)
- Связанные документы: `docs/PROJECT_RULES_FULL.md`, `reports/audit_conclusion_2026-02-20.md`.

## Верификация
- Ожидаемое поведение: нет секретных строк в отслеживаемых файлах и в отчётах; `.env` остаётся локальным и игнорируется Git.
- Доказательства: команды проверок + список найденных файлов (без содержимого).
