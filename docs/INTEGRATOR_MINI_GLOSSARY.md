# Мини-словарь integrator (термин → пример)

## Зачем этот словарь
- Быстро понять основные слова проекта без сложной теории.
- Использовать как шпаргалку для новых участников.

## Термины с примерами
- Монетизация → как проект начинает приносить деньги; пример: платный пакет “Readiness & Review Setup”.
- CLI → работа через команды в терминале; пример: `python -m integrator quality summary --json`.
- Quality gates → обязательные проверки качества; пример: `ruff`, `mypy`, `unittest` должны пройти.
- Preflight → быстрый стартовый health-check перед задачей; пример: `python -m integrator doctor`.
- Readiness → проверка готовности репозитория к безопасным изменениям; пример: `quality public-readiness`.
- Remediation plan → план исправлений при несоответствиях; пример: `ruleset_remediation_plan_*.json`.
- Probe-on-remote → безопасная проверка payload через временный ruleset в GitHub API.
- Shape-adapter → набор вариантов формы payload для совместимости API; пример: `checks_as_strings`.
- Compatible payload → вариант payload, который реально принимается API и готов к apply.
- Fail-fast → остановка сразу при опасной/невалидной ситуации; пример: `compatible_payload_missing`.
- Runbook → пошаговая инструкция действий; пример: цепочка `mismatch -> probe -> apply -> post-check`.
- Artifacts → файлы-доказательства выполнения; пример: JSON-отчёты в `reports/`.
- DevEx → удобство жизни разработчика; пример: понятные команды и предсказуемые проверки.
- Triage labels → метки приоритета/типа/статуса задач; пример: `priority:p1`, `status:in_progress`.
- Stale issue → задача, которая давно не обновлялась; пример: issue без апдейта более 7 дней.
- Projects migration readiness → сигнал, пора ли переходить из issue-only в GitHub Projects board.
- KPI → измеримый показатель прогресса; пример: `>=30%` снижение ручной рутины.
- CAC → стоимость привлечения клиента; пример: время и усилия на один пилот.
- LTV → ценность клиента за всё время; пример: регулярный retainer за поддержку.
- Churn → отток клиентов; пример: клиент ушёл, потому что не увидел пользы в цифрах.
- Retainer → регулярная абонентская оплата; пример: ежемесячный пакет операционной поддержки.
- Productized service → услуга, оформленная как стандартный повторяемый пакет; пример: “Integrator Bootstrap”.

## Как использовать словарь
- При чтении отчётов сначала сверять непонятные слова здесь.
- При написании новых docs использовать те же термины в том же значении.
