# Session Close Ops

## Когда вызывать

- Пользователь просит закрыть сессию по правилам.
- Нужен самоанализ по выполненным шагам и ошибкам.
- Нужен анализ качества пользовательских промптов и улучшения маршрутизации.
- Нужен финальный closure-отчёт с артефактами и подтверждёнными проверками.

## Когда не вызывать

- Нужен точечный фикс кода без closure-контекста.
- Нужен только security audit без postmortem.
- Нужна только эксплуатация LocalAI RAG/MCP без анализа сессии.

## Scope

- Session close по правилам проекта.
- Self-review: что сработало, что не сработало, какие риски остались.
- Prompt-review: паттерны запросов пользователя, где были потери точности, что улучшить.
- Рекомендации по оптимизации skill routing и agent orchestration.

## Рабочий цикл

1) Собрать факты из команд, логов и git-состояния.
2) Сформировать самоанализ действий и решений.
3) Сформировать анализ промптов пользователя: сильные стороны, слабые стороны, улучшения.
4) Проверить нужность новых skills/агентов и добавить только при измеримом выигрыше.
5) Выпустить closure-отчёт с ссылками на артефакты и следующими шагами.

## Quality Gate

- Для skill-изменений обязательно прогнать `python -m tools.check_skills_sync --json`.
