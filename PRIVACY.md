# PRIVACY

`integrator` — локальный CLI-инструмент. По умолчанию он работает с локальными файлами и запускает внешние процессы только по явной команде пользователя (например, `integrator run ...`).

## Какие данные обрабатываются
- Локальные пути, переданные через аргументы CLI (`--roots`, `--cwd` и т. п.), и файлы, найденные при сканировании.
- Git-метаданные локальных репозиториев (ветка, счётчики изменений, URL origin remote).
- Контент, который пользователь явно передаёт командам (например, файлы с текстом для операций).
- URL сервисов, которые пользователь явно задаёт (например, для проверок доступности).

## Какие данные сохраняются локально
- Отчёты и артефакты под `reports/` (JSON/MD диагностика и логи).
- Временные артефакты в каталогах OS temp при тестах и в некоторых утилитах.
- Секреты не записываются в файлы, предназначенные для коммита. Токены хранятся локально в `.env` или token-файлах и не коммитятся.

## Сетевые запросы
Сетевые запросы выполняются только в отдельных командах/модулях:
- GitHub API (`https://api.github.com/...`) для проверок репозитория и операций с Issues. Аутентификация использует `Authorization: Bearer <token>` при наличии токена.
- MOEX ISS (`https://iss.moex.com/...`) для экспорта рыночных данных.
- Локальные HTTP-проверки доступности сервисов (например, `/health`, `/v1/models`) для RAG и LM Studio.
- Шлюз Agent Memory: `POST /agent/memory/write` на настроенном base URL при использовании записи memory.

## Токены и учётные данные
- GitHub токен читается из `GITHUB_TOKEN` / `GH_TOKEN`, `GITHUB_TOKEN_FILE` / `INTEGRATOR_GITHUB_TOKEN_FILE` или локального `.env`.
- Токен Agent Memory передаётся как явное значение конфигурации/CLI и уходит только в заголовке `Authorization`.
- Токены не печатаются. В отчётах используется только булево состояние наподобие `token_present`.

## Контроль со стороны пользователя
- Оффлайн-режим достигается отсутствием токенов и отказом от команд, которые используют сетевые функции.
- Историю артефактов можно удалить локально, очистив содержимое `reports/`.
- Ротация токенов выполняется локально через обновление `.env` или token-файлов.
